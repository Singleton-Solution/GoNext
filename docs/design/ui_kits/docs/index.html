Installation.

Three ways to get GoNext running — cloud, CLI, or self-hosted. The cloud option takes about ninety seconds.

Cloud (recommended).

The fastest path. Go to gonext.app, sign in, and pick a starting template. We provision a Postgres database, an S3 bucket, and a Next.js edge runtime — all on the free tier.

You'll get a yoursite.gonext.app subdomain immediately. Add a custom domain whenever you're ready.

Free tier limits

10k monthly visits, 5 GB media, one site. No credit card. Upgrade lifts the visit cap and unlocks custom domains.

Command line.

Prefer the terminal? Install the CLI and run gonext init. Requires Go 1.22+ (we'll install Next.js for you).

bash
# macOS / Linux
brew install gonext/tap/gonext

# or via Go
go install github.com/gonext/gonext@latest

# scaffold a new site
gonext init my-site --template="broadsheet"
cd my-site
gonext dev    # http://localhost:3000

What the CLI sets up

  • A gonext.toml config at the project root
  • A Postgres-shaped SQLite db for local dev (no Docker required)
  • Next.js 15 with the GoNext runtime preinstalled
  • A themes/ directory you can fork and customize

Self-host.

One binary, one container, one Postgres database. We publish a Helm chart for Kubernetes and a Compose file for everyone else.

yaml · docker-compose.yml
services:
  gonext:
    image: ghcr.io/gonext/gonext:1.0
    ports: ["3000:3000"]
    environment:
      DATABASE_URL: postgres://...
      STORAGE_URL:  s3://my-bucket
Heads up — Postgres version

GoNext requires Postgres 14 or higher. Older versions will boot but full-text search will silently degrade.

What's next.

Once GoNext is running, the most-asked next steps are:

  • Write your first post in the block editor
  • Import an existing WordPress site via the WXR importer
  • Connect a custom domain (Cloud) or set up TLS (Self-host)
